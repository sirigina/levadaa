# Le Vada — Claude Code Context

## What this is

A satirical single-page React app. User uploads a photo → app "analyzes" it against eight dubious doctrines (FWHR, 2D:4D, jaw, grooming, V-ratio, gaze, dog detection, ambient context) → produces a 1920s-style "Institute" certificate with a performance score → user can share it as a PNG via the native mobile share sheet.

**The joke is the deadpan voice.** Early-20th-century European research institute, fake doctors (Dr. Ambrose Vernier, founded 1923), mock journals. Keep that register in any copy changes — no winks to the camera, no modern slang. The satire tag at the bottom of the certificate is the only admission that it's a joke.

## Core constraints (do not break)

- **Everything runs client-side after load.** No backend. No API calls. No photo ever leaves the device. If a change would require a server, flag it first.
- **Deterministic scoring.** The SHA-256 of the file seeds the PRNG. Same photo → same score, always. Don't introduce `Math.random()` into scoring logic.
- **Mobile-first.** Primary target is iOS Safari and Chrome on Android. Desktop is fine but secondary.
- **Analysis screen is a fixed-viewport layout** (`position: fixed; height: 100dvh`). Photo + terminal readout must both be visible simultaneously — no page scrolling during the scan.
- **Share uses `navigator.share({ files })`** with a PNG generated by `html-to-image`. Fallback to download if unsupported.
- **localStorage only** for persistence (key: `le-vada:history:v1`, max 12 entries, 160px JPEG thumbnails).

## File map

- `src/main.jsx` — React entry, nothing interesting
- `src/App.jsx` — all screens (Intro, Upload, Analysis, Result) + Certificate + share logic. One file on purpose.
- `src/theories.js` — the eight doctrines, classifications, `mulberry32` PRNG, `generateCaseId`
- `src/styles.css` — CSS custom properties at top, sections clearly commented
- `index.html` — Google Fonts (Fraunces, JetBrains Mono, Newsreader) preloaded
- `vercel.json` — explicit Vite config for Vercel

## Design system

- **Fonts**: Fraunces (display, italic serif), Newsreader (body serif), JetBrains Mono (data/terminal)
- **Palette**: cream paper (`--paper: #efe6d2`), oxblood accent (`--accent: #7a1c2c`), deep ink, muted gold. All defined as CSS custom properties at the top of `styles.css` — change them there, don't hardcode colors inside components.
- **Motion**: CSS-only where possible. Scan line, stamp drop-in, count-up on score, staggered theory bar fills.

## Things people typically ask for — and how

- **Add a new doctrine**: append to the `THEORIES` array in `theories.js`. Keep the `{ key, name, source, measure: (rng) => string, blurb: (score) => string }` shape. Then add a matching line to the "Registered Methods of Analysis" list in the Intro in `App.jsx`.
- **Change the score classification bands**: `CLASSIFICATIONS` array in `theories.js`. Keep them monotonically increasing by `min`.
- **Tweak distribution spread**: `amped = 50 + (avg - 50) * 1.45` in `App.jsx`'s `handlePhoto`. Higher multiplier = more extreme results.
- **Different aesthetic**: edit CSS custom properties at the top of `styles.css`. Don't hunt through rules for color literals.

## Before you change something

- If it's a UI-only change, just do it.
- If it would touch scoring, persistence format, or the share pipeline, summarize the plan first — those are the load-bearing parts.
- Always verify mobile (narrow viewport) still looks right. The scan screen is the most likely to break.

## Running

```
npm run dev    # local with hot reload
npm run build  # production build to dist/
```

Deploy is via Vercel (Vite framework auto-detected) or `npx vercel --prod`.
